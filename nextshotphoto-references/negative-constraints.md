# 运镜感知的负面约束引擎

不同运镜组合下，负面约束需要自动调整——手持运镜下「不要晃动」是矛盾的，极特写下「手指形态自然」比中景更关键。每镜的负面约束不是简单继承上一镜的 list，而是根据当前运镜+景别+动作计算。

---

## 约束增益/衰减规则

| 运镜/景别 | 影响的负面约束 | 操作 | 原因 |
|-----------|---------------|------|------|
| 手持晃动 | 「构图平稳」「不要抖动」 | 🗑️ 删除 | 手持=抖动是特征不是缺陷 |
| 极特写 | 「手指形态自然」「不要多余手指」 | 🔺 增强 | 特写放大手指，失败概率倍增 |
| FPV 无人机 | 「产品轮廓保持清晰」 | ➕ 新增 | 动态模糊会吞掉产品细节 |
| 快速变焦/Snap Zoom | 「定格画面不要模糊」 | ➕ 新增 | 骤停时刻需要绝对清晰 |
| 荷兰角 | 「地平线保持水平」 | 🗑️ 删除 | 倾斜就是荷兰角的目的 |
| 子弹时间 | 「不要定格动作」 | 🗑️ 删除 | 定格是子弹时间的核心效果 |
| 环绕运镜 | 「人物背面/侧面角度正常」 | ➕ 新增 | 环绕必然出现非正面视角 |
| 慢动作 | 「不要运动模糊」 | 🗑️ 删除 | 慢动作可能伴随轨迹模糊 |
| 人物手持产品 | 「产品不要出现在非指定位置」 | ➕ 新增 | 手部互动时产品位置易偏移 |
| 多人交互 | 「前景背影与主体分离清晰」 | ➕ 新增 | 多人时前景遮挡易混淆 |

---

## 实现方式

在推导流程 Step 6（映射检查）之后、Step 7（组装）之前插入约束计算步骤：

```
Step 6.5: 计算运镜感知的负面约束
  negative_constraints = inherit_from_previous()  # 继承上一镜基础约束
  
  FOR each rule in NEGATIVE_CONSTRAINT_TABLE:
      IF rule matches current camera_movement + shot.scene + shot.action:
          APPLY rule.operation to negative_constraints:
              case "🗑️ 删除": remove constraint from list
              case "🔺 增强": increase priority of constraint
              case "➕ 新增": add constraint to list
  
  # 去重：相同含义的约束只保留最强的一条
  negative_constraints = deduplicate(negative_constraints)
```

## 示例

```
上一镜负面约束：
  不要塑料皮肤，不要手指畸形，不要多余肢体
  不要大幅扭头，不要真实品牌logo，构图平稳

当前运镜：手持晃动 + 人物手持咖啡杯
→ 自动调整：
  删除「构图平稳」（手持时抖动正常）
  新增「产品在手中位置稳定，不因晃动脱手」
  新增「咖啡杯内液体不溢出」
  其他约束继承不变

输出负面约束：
  不要塑料皮肤，不要手指畸形，不要多余肢体
  不要大幅扭头，不要真实品牌logo
  产品在手中位置稳定，不因晃动脱手       # 手持特有
  咖啡杯内液体不溢出                      # 手持特有
```

## 默认负面约束（每镜继承）

以下约束在所有运镜下默认保留，除非被规则表删除：

- 不要塑料皮肤 / 蜡像感
- 不要手指畸形 / 多余肢体
- 不要大幅扭头（保护脖子和肩膀）
- 不要真实品牌 logo
- 不要因为景别变化改变产品颜色和花纹
- 不要出现不存在的第二只手

根据具体场景补充针对性负面（如柔雾棚拍加「不要硬光」，户外加「不要死白天空」）。

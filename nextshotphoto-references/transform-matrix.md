# 运镜变换矩阵

每个运镜类型定义一组字段级变换规则。推导流程 Step 3 加载对应运镜的变换规则，对 shot 字段应用变换。

---

## 基础运镜

### 推镜头 (Push In)

```
变换规则：
  shot.scene:      中景 → 近景（景别向主体推进一级，压迫感增强）
  foreground:      因镜头推进而放大虚化，占据更多画面
  background:      被压缩裁切，部分元素出画
  depth:           景深变浅，背景虚化增强，焦点更集中
  composition:     主体周围空间减少，环境信息减弱
运镜描述短语：    "镜头平滑向前推进，从[首帧景别]推至更近"
映射检查：
  - ⚠️ 产品特写时确认产品不被裁切出画
  - ⚠️ 推进幅度过大可能丢失前景元素
```

### 拉镜头 (Pull Out)

```
变换规则：
  shot.scene:      近景 → 中景（景别向环境方向退一级）
  foreground:      因拉远而缩小，更多环境元素入画
  background:      更多空间显现，环境感增强，视野扩展
  depth:           景深变大，背景清晰度提高
  composition:     主体周围空间增大，环境信息增多
运镜描述短语：    "镜头平滑向后拉远，从[首帧景别]拉出展现更多环境"
映射检查：
  - ⚠️ 新入画元素不应与锁定域冲突（出现不该出现的东西）
```

### 环绕 (Orbit)

```
变换规则：
  shot.angle:      平视正对 → 移动到主体侧面/斜后方
  foreground:      环绕过程中前景物体从一侧到另一侧
  background:      根据环绕角度完全改变（从 A 场景转到 B 场景）
  lighting:        !! 需要检查——原锁定光源方向可能随视点变化失效
  composition:     主体从正面展示变为侧面/背面展示
运镜描述短语：    "镜头围绕主体做[半圆/全圆]平滑环绕运动"
映射检查：
  - ⚠️ 光线映射：环绕时逆光方案可能失效，需补侧面光
  - ⚠️ 人物正面特征不可见时，确认是否接受侧脸/背面
```

### 手持晃动 (Handheld)

```
变换规则：
  stability:       固定/平稳 → 轻微呼吸晃动，自然防抖
  composition:     画面有随机轻微偏移，非固定构图
运镜描述短语：    "手持拍摄风格，镜头带有自然轻微呼吸抖动"
映射检查：
  - 手持与极特写组合时抖动感需减弱（太近时晃得看不清）
```

### 摇镜头 (Pan)

```
变换规则：
  shot.angle:      注意力从左到右/从右到左扫视
  foreground:      镜头扫过时前景物体形成动态模糊
  background:      从场景 A 部分过渡到场景 B 部分
运镜描述短语：    "镜头水平向右/向左缓慢摇动，逐步揭示新空间"
映射检查：
  - ⚠️ 摇镜头以「揭示」为目的，需确认新区域无穿帮
```

### 跟随 (Follow)

```
变换规则：
  action:          人物向[方向]移动，镜头保持同步速度跟随
  foreground:      移动过程中前景连续变化
  background:      随人物移动连续变化的空间序列
运镜描述短语：    "镜头与主体保持同步速度，跟随人物向[方向]移动"
映射检查：
  - ⚠️ 移动路径需连续，不能跳跃空间
```

### 移镜头 (Track/Truck)

```
变换规则：
  shot.angle:      机位平行移动，不改变视角方向
  foreground:      平移过程中前景连续滑动
  background:      背景平行滑动，空间关系保持
运镜描述短语：    "镜头平行向[左/右]移动，平稳叙事"
映射检查：
  - ⚠️ 移镜头适合展示行走过程，不适合静态场景
```

### 升降镜头 (Pedestal)

```
变换规则：
  shot.angle:      升：从低向高拔起 / 降：从高向低落
  foreground:      升降过程中前景高度连续变化
  background:      升：展现更多远景 / 降：聚焦地面细节
运镜描述短语：    "镜头垂直[上升/下降]，[展现宏大场景/聚焦落点]"
映射检查：
  - 升镜头适合开场，降镜头适合收束
```

## 导演级运镜

### 希区柯克变焦 (Dolly Zoom)

```
变换规则：
  subject_size:    主体在画面中大小不变（核心特征）
  background:      因推拉+变焦的共同作用，背景被强烈压缩/拉伸
  emotion:         空间扭曲感，心理压迫，震惊效果
运镜描述短语：    "进行希区柯克变焦，摄像机后拉同时长焦推进，主体保持大小"
映射检查：
  - 此为强风格化运镜，运镜本身即为画面主题，需用户确认是否匹配镜头功能
```

### 无人机穿越 (FPV Drone)

```
变换规则：
  shot.angle:      固定机位 → 自由飞行视角，第一人称
  foreground:      高速移动中前景急速掠过，产生强烈动态模糊
  background:      连续穿越空间，背景快速变化，空间感强烈
  depth:           景深较大，远近皆清晰，增强临场感
  stability:       剧烈晃动，模拟无人机飞行姿态
运镜描述短语：    "FPV 无人机视角，高速穿梭，大幅度滚转与甩尾运镜，动态模糊"
映射检查：
  - ⚠️ FPV 视角下人物面部不可见，人物锁定字符仍需维持
  - ⚠️ 动态模糊过强时产品细节丢失，需在负面约束中补充「产品轮廓保持清晰」
```

### 子弹时间 (Bullet Time)

```
变换规则：
  scene:           动态 → 瞬间凝固定格
  action:          运动过程 → 时间近乎凝固，周围环境静止
  background:      背景中的人物/物体/粒子保持静止
  camera:          镜头在凝固的时空中缓慢移动/环绕
运镜描述短语：    "画面突然进入子弹时间（Bullet Time），时间近乎凝固，镜头在静止空间中缓慢移动"
映射检查：
  - ⚠️ 子弹时间是强风格化效果，需确认是否匹配镜头叙事功能
  - ⚠️ 人物表情在凝固瞬间被放大，需确保表情自然
```

### 快速变焦 (Snap Zoom)

```
变换规则：
  shot.scene:      中景/近景 → 瞬间推进至极特写
  action:          常规动作 → 在变焦终点处突然定格
  focus:           画面瞬间切换焦点到[指定部位/道具]
  emotion:         冲击感，强调，惊吓
运镜描述短语：    "极速光学变焦推镜头，在[焦点对象]处突然定格"
映射检查：
  - ⚠️ 瞬间变焦模式下产品不能出现在画面边缘（会被裁切）
  - 极特写时需加强手指负面约束
```

### 闪摇 (Whip Pan)

```
变换规则：
  shot.angle:      场景 A → 极速水平扫过 → 场景 B
  foreground:      扫过时产生强烈横向动态模糊
  background:      从一个空间切换到另一个空间
  motion_blur:     全程模糊，仅起点和终点清晰
运镜描述短语：    "镜头快速水平甩动，产生合理的运动模糊，从[场景A]切换到[场景B]"
映射检查：
  - ⚠️ 闪摇的终点场景需用户指定，不能默认「回到原位」
  - ⚠️ 闪摇不适合室内狭小空间（横向距离不够）
```

### 荷兰角 (Dutch Angle)

```
变换规则：
  shot.angle:      水平 → 向一侧倾斜 15-30 度
  composition:     地平线打破，画面失衡，不安感
  emotion:         心理失衡，诡异，紧张
运镜描述短语：    "画面呈现荷兰式倾斜（Dutch Angle），镜头向一侧倾斜[角度]度"
映射检查：
  - ⚠️ 荷兰角不适用于产品展示（会让产品看起来歪斜）
  - ⚠️ 建议超过 30 度会过于刻意，20 度为通用安全值
```

## 组合运镜

### 组合序列 (Multi-Shot Sequence)

```
变换规则：
  sequence:        [运镜1] + [运镜2] + [运镜3] 串联
  scene:           随每个子运镜分段变化
  action:          随序列推进逐步进展
  composition:     首帧 → 过渡帧序列 → 尾帧
运镜描述短语：    "多镜组合序列：[运镜1描述] → [运镜2描述] → [运镜3描述]"
映射检查：
  - ⚠️ 组合序列需用户指定子运镜顺序和时长占比
  - ⚠️ 序列长度不宜超过 3 个子运镜，否则模型难以稳定跟随
```

### 记忆闪回 (Memory Flashback)

```
效果：回忆感满格，边缘模糊+暖黄调+叠化
序列：        边缘模糊 → 暖黄调强化 → 3 秒叠化转场
变换规则：
  action:          当前动作 → 渐入回忆状态
  color:           当前色调 → 暖黄/褪色
  edge:            清晰 → 边缘模糊/柔化
  composition:     硬切 → 叠化过渡
运镜描述短语：    "画面边缘逐渐模糊，色调转为暖黄，通过3秒叠化进入回忆场景"
映射检查：
  - ⚠️ 需要确认闪回的是哪个人物/时间点
  - ⚠️ 闪回结束后需要回到当前时间线
```

### 空间折叠 (Space Fold)

```
效果：穿越次元，画面折叠+光线扭曲
序列：        画面折叠 → 光线扭曲 → 新空间展开
变换规则：
  composition:     正常空间 → 折叠/扭曲 → 新空间
  lighting:        当前光线 → 光线扭曲/折射 → 环境光
  emotion:         正常 → 奇幻/错乱
运镜描述短语：    "画面像纸一样折叠，光线随之扭曲，展开时已进入新的空间"
映射检查：
  - ⚠️ 强风格化效果，需确认是否匹配镜头叙事功能
```

### 时间凝滞 (Time Freeze)

```
效果：高速骤停+2s定格，定格震撼
序列：        正常速度 → 骤停 → 2 秒定格 → 恢复
变换规则：
  action:          运动 → 瞬间定格，所有动态停止
  focus:           骤停瞬间焦点锁死
  emotion:         冲击感，强调
运镜描述短语：    "高速运动画面骤然停止，定格2秒，所有动态凝固"
映射检查：
  - ⚠️ 定格时人物表情被放大，需确保自然
  - ⚠️ 定格结束后动作不能跳跃（需自然接续）
```

### 疾速追尾 (Chase Tail)

```
效果：追逐代入，跟拍3-5m+惯性延迟
序列：        跟拍追尾 → 惯性延迟 → 贴近主体
变换规则：
  shot.angle:      跟拍追尾视角，镜头与主体保持3-5m距离
  foreground:      追逐路径上的障碍物急速掠过
  background:      背景连续高速流动
  motion_blur:     全程强烈动态模糊
运镜描述短语：    "镜头紧贴主体后方跟拍，保持3-5米追尾距离，主体有轻微惯性延迟"
映射检查：
  - ⚠️ 追尾视角下人物面部不可见，动作主靠背影和肢体表达
  - ⚠️ 产品在追逐场景中需要声明保持可见
```

### 飞跃俯冲 (Leap Dive)

```
效果：上帝降临，俯冲+70%拉起+速度线
序列：        高空俯视 → 极速俯冲 → 70%拉起 → 平飞
变换规则：
  shot.angle:      高空俯视 → 俯冲接近 → 拉起平飞
  foreground:      俯冲时地面急速放大
  background:      高空俯视全景 → 俯冲时地面细节涌现
  depth:           全程大景深，远近清晰
运镜描述短语：    "镜头从高空俯视极速俯冲，接近地面时70%拉起，带速度线效果"
映射检查：
  - ⚠️ 俯冲终点需明确（拉起到什么位置？降落点？）
  - ⚠️ 不适合室内场景（垂直距离不够）
```

## 动作驱动的字段覆盖

当用户指定了动作（action），动作描述覆盖运镜变换中的动作默认值。运镜变换先决定"机位怎么动"，动作描述再决定"人物/产品怎么变"。

```
输入：
  运镜 = 推镜头 (Push In)
  动作 = "人物拿起 @coffee_cup，低头闻香"

推导步骤：
  ① 运镜变换：景别 中景 → 近景，背景压缩，景深变浅
  ② 动作覆盖：shot.action = "人物拿起咖啡杯，低头闻香"
     → 自动识别 @coffee_cup，更新 products[0].interaction = "人物右手握着"
     → 其他产品（如 @book）的 interaction 保持不变
  ③ 映射检查：产品从「放在木托盘上」变为「人物手中」，已覆盖
```

## 完整推导流程

```
Step 1: 复制锁定域（不变）
  tail.lock = first_context.lock  — 逐字符复制

Step 2: 复制当前镜头域
  tail.shot = first_context.shot

Step 3: 应用运镜变换矩阵
  FOR each rule in CAMERA_MOVEMENT_TABLE[camera_movement].transforms:
      IF rule.field matches shot field:
          tail.shot[field] = apply(first_context.shot[field], rule)

Step 4: 如果用户指定了动作，覆盖 action 字段
  IF shot_action is not empty:
      tail.shot.action = shot_action
      CALL update_product_interaction(shot_action) — 更新 @产品ID 对应的 interaction

Step 5: 运镜描述组装
  motion_phrase = CAMERA_MOVEMENT_TABLE[camera_movement].motion_desc
  → 植入 prompt 变体域

Step 6: 映射检查
  FOR each check in CAMERA_MOVEMENT_TABLE[camera_movement].mapping_checks:
      IF check triggers on current state:
          ADD to issues list

Step 6.5: 计算运镜感知的负面约束
  见 references/negative-constraints.md

Step 7: 组装完整 prompt + 输出映射报告
  prompt = assemble(tail, motion_phrase, negative_constraints)
  output = prompt + mapping_report + updated_context
```

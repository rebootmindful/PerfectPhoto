写了一个 AI 写真 prompt 生成器，叫 PerfectPhoto。

花了一天，从评估改到发布。中间出了 10 张验证图，跑了 ImageGen 和 DragonCode 两条管线，全链路走通。

说说它解决什么问题。

你写一句「好看的美女」，AI 给你一张空洞的漂亮脸。加「氛围感、电影感、高级感」，AI 还是不知道往哪使劲。prompt 太虚的时候，模型使不上劲。

同类 prompt 生成器，要么堆元素库（skill-prompt-generator），要么堆 prompt 数量（awesome-nano-banana）。没有人在乎用户想要什么感觉。

所以我加了几个东西。

用户说「逝去的青春」，拆成 7 个维度：年代、色调、光影、道具、情绪、氛围、人物气质。后续 8 步全部按这个方向裁切。

用户说「想要那种味道」，给了 13 位摄影大师预设。选了上田义彦，成像自动对齐大画幅，光线对齐低饱和。选了王家卫，光线那步自动优先霓虹光溢出。选了大师，参数自动跟。

用户说要一组图，我做了人物锁定。把脸型、发型、妆容、服装抽出来冻成一块，每个变体字面量整块植入。测了 3 个变体，人物衣服颜色没漂。

从同行学了几项。用途前置代替空泛形容词——第一行写死用途（小红书封面/朋友圈/公众号/打印输出），一行锁死用途比五句风格描述管用。每份 prompt 至少删一个形容词——写完问自己「这个词去掉后画面有不同吗？」没有就删。用户说「像王家卫电影」时先搜一下视觉特征再写导演笔记，不靠记忆瞎猜。镜头焦段、大师风格、胶片模拟、布光方向、光圈景深各一张速查表，用户犹豫时 agent 自动搬出来推荐。

今天跑了两个完整场景。夏日花田，快速模式 3 步命中一次出图。夕阳旧梦，一男一女在田间跳舞，从意图编译到出图到 3 个变体，7 张全通。公子如玉山间泉水，选了上田义彦大师，走了人物锁定和导演笔记，冒险度选安全，DragonCode 一次出图。7 次验证 7 次成功。

但范围窄。都是写真，不是产品图 UI mockup 教学插图。缺雨天雪景赛博家居。诊断模式写完了但零实测，这是我最想测的——谁有空扔个 prompt 过来试试。

然后我又做了一件延伸的事。

用户把写真做好了，要拍视频——每换一个镜头，人物/产品/氛围都要重新写一遍 prompt，写慢了漂移。所以我加了个分镜生产引擎进去，叫 NextShotPhoto。

不走 8 步，只问变化量。用户说「下一个镜头推到产品特写」，我拆成 4 个 Delta（景别/机位/运镜/动作），缺什么问什么。然后跑运镜变换矩阵——推镜头就自动让景别变近、背景压缩、景深变浅。每个运镜（推/拉/环绕/手持/摇/跟随/FPV/子弹时间/快速变焦/闪摇/荷兰角）都有硬编码的字段级变换规则，不走模型推理，不漂移。

还做了大师运镜库。选了斯皮尔伯格，环绕自动变成半圆+移焦。选了诺兰，推镜变成 IMAX 慢推。王家卫只签名手持，不签名推镜——这就是品味。

每镜输出两个东西：尾帧图片 prompt（给 ImageGen）和视频生成提示词（给 Seedance）。用户拿到首帧图 + 尾帧图 + 视频提示词，喂给视频模型，完事。

项目还是同一个，在 github.com/rebootmindful/PerfectPhoto。对 agent 说「帮我写个写真 prompt」或「做下一个镜头」就能跑。
